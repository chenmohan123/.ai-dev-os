# 移动端项目交付清单模板

## 目标

- 提供一份可直接套用的移动端项目交付清单，用于串联开发、自测、真机验收、出包、发布、灰度和回滚准备。
- 避免项目在准备上线时，仍通过口头确认或零散消息核对关键交付项。
- 让移动端交付动作从“靠经验记忆”转为“按清单执行”。

## 适用范围

- 适用于 Flutter、React Native、原生 iOS、原生 Android 及混合移动应用工程的版本交付、自测出包、灰度发布、正式上线和回滚准备场景。
- 涉及发布交付要求时，应结合 [15_移动端发布交付规范](15_移动端发布交付规范.md) 使用。
- 涉及测试验收要求时，应结合 [16_移动端测试与验收规范](16_移动端测试与验收规范.md) 使用。
- 涉及原生能力上线影响时，应结合 [14_移动端原生能力接入规范](14_移动端原生能力接入规范.md) 使用。

## 使用说明

- 本模板适合作为版本发布前的执行清单。
- 可按项目实际情况补充“是否适用”“负责人”“完成时间”“备注”等列。
- 不适用项应写明原因，不直接留空。

## 模板

### 1. 基本信息

- 版本号：
- 构建号：
- 目标渠道：正式版 / 灰度版 / 测试版 / 内测版 / 企业分发版
- 目标平台：iOS / Android / 双端
- 构建基线：
- 发布负责人：
- 验收负责人：
- 计划发布时间：

### 2. 开发完成检查

- [ ] 需求范围已冻结，未完成项已明确记录。
- [ ] 目标平台、渠道和环境边界已确认。
- [ ] 原生能力、权限弹窗、深链和推送改动边界已确认。
- [ ] 相关配置变更、渠道开关和环境变量已确认。
- [ ] 发布说明、变更说明或用户提示已补齐。

### 3. 自测与测试检查

- [ ] 单元测试已通过。
- [ ] 集成测试或关键模块协作测试已通过。
- [ ] 关键冒烟路径已验证。
- [ ] 真机最小验收路径已验证。
- [ ] 安装后启动、升级安装和回退场景已验证。
- [ ] 本轮已知问题已汇总并确认是否阻断发布。

### 4. 原生能力检查

- [ ] 相机 / 相册相关能力已验证或确认不适用。
- [ ] 定位 / 地图相关能力已验证或确认不适用。
- [ ] 推送 / 通知 / 深链相关能力已验证或确认不适用。
- [ ] 文件导入导出 / 分享 / 生物识别等相关能力已验证或确认不适用。
- [ ] 权限拒绝、用户取消和系统不可用场景已验证或确认不适用。

### 5. 构建与制品检查

- [ ] 版本号、构建号、渠道标识和目标平台信息正确。
- [ ] 安装包、商店包、企业分发包或灰度包已生成。
- [ ] 签名状态已验证。
- [ ] 制品命名和归档路径符合规范。
- [ ] 对应构建日志和测试结果已归档。

### 6. 配置与环境检查

- [ ] API 地址、推送环境、深链配置和功能开关符合目标环境。
- [ ] 渠道差异配置与环境区分已确认。
- [ ] 敏感配置、签名材料和发布凭证未进入普通日志或制品。
- [ ] 开发、测试、灰度、正式环境未混用。

### 7. 发布前验收检查

- [ ] 安装包可安装并正常启动。
- [ ] 登录链路和核心主流程可用。
- [ ] 推送点击、深链落点和返回路径符合预期或确认不适用。
- [ ] 原生能力上线影响已完成最小验收。
- [ ] 数据迁移、缓存保留和权限变化行为符合预期。
- [ ] 关键页面或关键流程已完成真机体验验收，已确认主任务焦点、信息层级和操作反馈达标。
- [ ] 未残留明显默认组件样式感、桌面式缩放布局或只满足“能点能提审”的页面质量问题。

### 8. 发布执行检查

- [ ] 发布窗口时间已确认。
- [ ] 发布负责人、观察人和回滚负责人已到位。
- [ ] 商店提审、企业分发、灰度扩量或渠道上架步骤已确认。
- [ ] 灰度样本范围、停推条件和放量规则已确认。

### 9. 回滚准备检查

- [ ] 上一稳定版本制品可用。
- [ ] 商店回退、灰度停止或企业分发替换方式已确认。
- [ ] 配置回滚、服务端开关降级和数据恢复边界已确认。
- [ ] 故障升级联系人和通知路径已确认。

### 10. 发布后观察检查

- [ ] 启动率、崩溃率、关键接口错误率和核心业务指标已开始观察。
- [ ] 推送点击、深链落点和关键原生能力相关异常已重点观察。
- [ ] 用户反馈入口已准备。
- [ ] 发布结果和后续行动项已记录。

## 输出要求

- 输出交付清单时，应保留完整执行项，不随意压缩为口头描述。
- 如项目已有内部发布单格式，可在保留本模板核心检查项的前提下适配现有格式。
- 如某项检查由自动化系统完成，也应记录对应结果来源。
- 如版本包含页面重做、关键流程改版或演示场景，还应附真机截图、录屏或等价体验验收材料。
