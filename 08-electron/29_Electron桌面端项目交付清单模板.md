# Electron 桌面端项目交付清单模板

## 目标

- 提供一份可直接套用的 Electron 桌面端项目交付清单，用于串联开发、自测、打包、验收、发布和回滚准备。
- 避免项目在准备上线时，仍通过口头确认或零散消息核对关键交付项。
- 让交付动作从“靠经验记忆”转为“按清单执行”。

## 适用范围

- 适用于 Electron 桌面端的版本交付、自测出包、灰度发布、正式上线和回滚准备场景。
- 涉及测试验收要求时，应结合 [25_Electron桌面端测试与验收规范](25_Electron桌面端测试与验收规范.md) 使用。
- 涉及打包、签名、制品和发布链路时，应结合 [24_Electron桌面端打包发布与签名规范](24_Electron桌面端打包发布与签名规范.md) 使用。
- 涉及流水线阶段和自动发布编排时，应结合 [27_Electron桌面端CI_CD与流水线规范](27_Electron桌面端CI_CD与流水线规范.md) 使用。
- 涉及配置、环境变量和渠道开关时，应结合 [26_Electron桌面端配置与环境管理规范](26_Electron桌面端配置与环境管理规范.md) 使用。

## 使用说明

- 本模板适合作为版本发布前的执行清单。
- 可按项目实际情况补充“是否适用”“负责人”“完成时间”“备注”等列。
- 不适用项应写明原因，不直接留空。

## 模板

### 1. 基本信息

- 版本号：
- 目标渠道：正式版 / 灰度版 / 测试版 / 内测版
- 目标平台：Windows / macOS / Linux
- 构建基线：
- 发布负责人：
- 验收负责人：
- 计划发布时间：

### 2. 开发完成检查

- [ ] 需求范围已冻结，未完成项已明确记录。
- [ ] 主进程、preload、渲染层改动边界已确认。
- [ ] 相关配置变更、渠道开关和环境变量已确认。
- [ ] 新增原生能力、IPC 或高权限调用已完成安全复核。
- [ ] 文档、变更说明或用户提示已补齐。

### 3. 自测与测试检查

- [ ] 单元测试已通过。
- [ ] IPC 或集成测试已通过。
- [ ] 关键冒烟路径已验证。
- [ ] 安装后启动、升级安装和回滚场景已验证。
- [ ] 多窗口、托盘、通知、文件操作等关键桌面能力已验证。
- [ ] 本轮已知问题已汇总并确认是否阻断发布。

### 4. 构建与制品检查

- [ ] 版本号、渠道标识和目标平台信息正确。
- [ ] 安装包、更新包、符号文件和校验摘要已生成。
- [ ] 签名状态已验证。
- [ ] 制品命名和归档路径符合规范。
- [ ] 对应构建日志和测试结果已归档。

### 5. 配置与环境检查

- [ ] 更新源、API 地址、日志等级和功能开关符合目标环境。
- [ ] 敏感配置、签名材料和发布凭证未进入普通日志或制品。
- [ ] 正式、灰度、测试或内测渠道未混用。
- [ ] 运行时配置默认值和降级策略已确认。

### 6. 发布前验收检查

- [ ] 安装包可安装并正常启动。
- [ ] 主窗口可进入且核心主流程可用。
- [ ] 自动更新链路已按目标渠道验证。
- [ ] 数据迁移、配置保留和缓存清理行为符合预期。
- [ ] 发布验收记录已输出。

### 7. 发布执行检查

- [ ] 发布窗口时间已确认。
- [ ] 发布负责人、观察人和回滚负责人已到位。
- [ ] 发布步骤、审批节点和人工确认点已确认。
- [ ] 灰度样本范围、停推条件和放量规则已确认。

### 8. 回滚准备检查

- [ ] 上一稳定版本制品可用。
- [ ] 渠道停推或回退方式已确认。
- [ ] 配置回滚、更新回滚和数据恢复边界已确认。
- [ ] 故障升级联系人和通知路径已确认。

### 9. 发布后观察检查

- [ ] 关键日志、崩溃指标和错误反馈已开始观察。
- [ ] 用户反馈入口已准备。
- [ ] 高优先级问题的止血动作已预设。
- [ ] 发布结果和后续行动项已记录。

## 输出要求

- 输出交付清单时，应保留完整执行项，不随意压缩为口头描述。
- 如项目已有内部发布单格式，可在保留本模板核心检查项的前提下适配现有格式。
- 如某项检查由自动化系统完成，也应记录对应结果来源。
