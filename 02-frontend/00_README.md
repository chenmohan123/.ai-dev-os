# 目录索引

## 目标

- 为前端目录提供统一入口，先说明通用前端、管理后台页面、AI 应用前端、官网落地页、移动 Web 页面、技术选型和目录治理分别该从哪里进入。
- 避免把前端规范只理解成页面实现细节，而忽略页面类型判断、模板选择、技术选型和结构治理约束。

## 使用说明

- 进入前端规范目录后，默认先阅读 [01_通用前端规范](01_通用前端规范.md)，再根据场景进入管理后台通用规范、AI 应用前端分册、官网落地页分册、移动 Web 分册或具体技术栈分册。
- 如任务属于管理后台页面，默认继续阅读 [02_管理后台页面通用规范](02_管理后台页面通用规范.md)。
- 如任务需要直接生成后台页面骨架、判断页面类型或确定某类页面默认结构，应继续阅读 [03_管理后台页面模板规范](03_管理后台页面模板规范.md)。
- 如任务还不确定应该按管理后台、AI 应用前端还是官网落地页来理解，应先阅读 [13_前端页面场景选型规范](13_前端页面场景选型规范.md)。
- 如任务属于对话式 AI 应用、智能助手页、Agent 工作台或消息流交互页面，应继续阅读 [11_AntDesignX智能应用前端规范](11_AntDesignX智能应用前端规范.md)。
- 如任务属于官网、品牌页、产品介绍页、活动页或营销落地页，应继续阅读 [12_AntDesignLanding官网与落地页规范](12_AntDesignLanding官网与落地页规范.md)。
- 如任务需要直接生成 AI 应用页面骨架或官网落地页骨架，应继续阅读 [14_智能应用与官网页面模板规范](14_智能应用与官网页面模板规范.md)。
- 如任务属于业务 H5、移动 Web 工作台、移动表单页、扫码业务页或移动 Web AI 助手页，应继续阅读 [15_移动端页面场景选型规范](15_移动端页面场景选型规范.md) 和 [16_AntDesignMobile前端规范](16_AntDesignMobile前端规范.md)。
- 如任务要求页面具备更强品牌感、专业感、惊艳度或跨展示面统一视觉质量，应继续阅读 [18_前端高质量视觉输出规范](18_前端高质量视觉输出规范.md)。
- 如任务需要确定新项目、模板仓库或演示仓库的前端推荐版本，应继续阅读 [19_前端稳定版本基线清单](19_前端稳定版本基线清单.md)。
- 如任务主体是微信小程序或同类平台型轻应用，不留在本目录，返回 [../10-miniapp/00_README.md](../10-miniapp/00_README.md)。
- 如任务已明确为 Vue 或 React 管理后台，再进入对应技术栈分册。
- 如任务涉及 Vue 管理后台的新项目选型、已有项目方案判断或目录结构设计，应继续阅读 Vue 的配套分册，而不是只看页面规范。
- 如任务涉及 React 管理后台的新项目选型、已有项目方案判断或目录结构设计，应继续阅读 React 的配套分册，而不是只看页面规范。

## 什么时候先读本页

- 还不确定当前任务属于前端通用实现、后台页面规则、AI 应用前端、官网落地页、移动 Web 页面、模板选择、技术选型还是目录治理。
- 需要先判断当前页面应该按哪类后台页面模板生成，或该进入 Vue/React 哪组分册。
- 需要先判断当前页面应归入管理后台、AI 应用前端、官网落地页还是移动 Web 页面。
- 当前任务虽然是页面开发，但还没明确是否要补读后端、安全、测试或交付目录。

## 与其它目录的配合方式

- 前端目录负责回答“页面怎么组织、模板怎么选、技术栈怎么判断、结构怎么治理，以及 AI 应用前端、官网落地页和移动 Web 页面各自该用什么边界”。
- 本目录中的移动相关分册默认指移动 Web、H5 和 WebView 页面，不覆盖原生或跨端 App 工程。
- 本目录也不覆盖微信小程序及同类平台宿主内应用；此类任务统一转到 [../10-miniapp/00_README.md](../10-miniapp/00_README.md)。
- 涉及接口契约、数据模型和服务边界时，补读 [../03-backend/00_README.md](../03-backend/00_README.md)。
- 涉及权限显隐、敏感字段、上传下载和前端安全边界时，补读 [../07-security/00_README.md](../07-security/00_README.md)。
- 涉及页面验收、回归范围和发布门禁时，补读 [../05-tests/00_README.md](../05-tests/00_README.md) 和 [../01-general/08_交付质量规范.md](../01-general/08_交付质量规范.md)。
- 通用需求、命名、评审和语言层规则仍以 [../01-general/00_README.md](../01-general/00_README.md) 为基础。

## 不建议的做法

- 还没判断页面类型和模板边界，就直接开始堆列表、表单和弹窗实现。
- 把 Ant Design X 当成管理后台默认组件库，或把 Ant Design Landing 当成后台页面骨架方案。
- 把 Ant Design Mobile 当成所有 React 页面默认组件库，而不先判断是否真是移动 Web 主场景。
- 把技术选型、目录结构和页面实现混在同一轮里同时决策，导致依据不清。
- 只看前端目录，不补读后端、安全、测试和交付目录的相关约束。
