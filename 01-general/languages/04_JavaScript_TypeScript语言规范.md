# JavaScript_TypeScript语言规范

## 适用范围

- 适用于 JavaScript、TypeScript 前端项目、Node.js 程序和共享工具模块的实现层规范。

## 目标

- 让 JavaScript/TypeScript 代码在模块边界、类型表达、异步流程和状态管理上保持清晰一致。
- 避免 any 扩散、共享状态失控、环境边界混乱和临时代码上升为长期公共依赖。

## 基本要求

- 模块边界要清楚，页面逻辑、状态管理、服务调用和通用工具不要混在同一文件。
- TypeScript 优先显式建模关键数据结构，不靠 any、隐式推断和约定俗成维持复杂系统。
- 异步流程、错误处理和状态更新要有统一约定，避免回调、Promise 和副作用混杂。
- 公共工具抽取以稳定复用为前提，不把一次性页面逻辑过早下沉到 shared。

## 项目与模块组织要求

- 目录、模块和包边界优先围绕业务域、运行环境和职责层组织，而不是只按 components、utils、services 机械拆分。
- shared、common、utils、hooks 等公共目录只承载稳定复用能力，不持续吸收页面私有逻辑。
- 浏览器端、Node 端、构建脚本和共享契约应有稳定落点，避免在同一模块里混写多种运行场景。

## 类型与状态要求

- 关键接口、服务返回、状态对象和跨模块契约优先使用显式类型表达。
- any、as 强转和隐式可空仅在确有边界需要时使用，不作为默认开发捷径。
- 局部页面状态、全局状态和服务端状态边界应清楚，不让一个状态层兜底所有场景。
- 枚举值、事件名、错误码和配置键应集中管理，不散落在多个模块中用字符串硬编码。

## 契约与弱类型边界要求

- 对外接口、事件载荷、配置对象和跨模块公共契约优先使用清晰可演进的类型，而不是长期依赖 Record<string, any>、unknown 强转或弱类型对象透传。
- 服务端返回、表单模型、查询参数和路由参数的转换应有明确边界，不让弱类型数据在多个层级长期漂移。
- 可空值、默认值、空数组和未加载状态语义要明确区分，不用单一 undefined/null 表达所有状态。

## 实现约束

- 函数和组件名应表达实际职责，不用 handler、util、data 这类泛名掩盖意图。
- 优先通过模块拆分和清晰输入输出控制复杂度，而不是堆 if/else 和共享可变状态。
- 类型定义靠近使用场景维护，跨模块稳定契约再上收共享层。
- 浏览器环境、Node 环境和构建时逻辑要区分清楚，避免环境假设混乱。

## 配置与运行时边界要求

- 环境变量、运行时配置和构建时开关应集中在配置边界统一读取和校验，不在业务模块里零散访问 process.env、import.meta.env 或全局注入对象。
- 浏览器 API、Node API、构建工具上下文和测试运行时假设应隔离处理，不把环境判断散落在核心业务流程中。
- 框架特有 hooks、生命周期和副作用容器应用于边界协调，不把核心业务规则长期埋入框架回调链。 

## 异步与副作用要求

- Promise、async/await、事件回调和副作用触发方式应在同一项目中保持一致风格。
- 网络请求、缓存读取、订阅解绑和定时器清理应有明确边界，不把副作用散落到任意工具函数。
- 错误处理应保留关键上下文，不用空 catch 或只返回默认值掩盖失败。
- 浏览器端、Node 端和构建脚本中的副作用逻辑应分开维护，不互相假设运行环境。

## 常见风险信号

- any、unknown 强转和类型断言越来越多，说明类型模型没有真正收敛。
- shared、utils、hooks 或 services 目录持续吸收页面私有逻辑，说明公共边界失控。
- 一个模块同时管理视图状态、服务调用、副作用清理和数据转换，说明职责混杂。
- 同一项目里混用多套异步和状态管理风格，导致行为不可预测。
- 配置读取、环境判断和弱类型对象转换散落在各层，说明契约与运行边界没有真正收敛。

## 验收清单

- 是否存在 any 泛滥、共享状态失控或模块职责混杂。
- 是否让类型、异步和错误处理保持统一风格。
- 是否避免把临时代码和页面细节沉淀成长期公共依赖。
- 是否明确了公共契约、可空语义和运行时配置边界。

## 输出模板建议

- 模块边界：说明页面、状态、服务、共享工具和运行环境边界。
- 类型策略：说明关键契约、状态对象和错误模型如何建模。
- 异步策略：说明请求、副作用、错误处理和清理方式。
- 配置边界：说明环境变量、运行时配置和环境判断的集中管理方式。
- 风险与限制：说明 any、共享状态或环境混用的治理点。

## 不建议的做法

- 用 any、类型断言和隐式约定长期维持复杂模块。
- 把页面私有逻辑过早抽到 shared 或 utils 中长期复用。
- 在同一模块里混杂视图渲染、状态管理、网络请求和副作用清理。
- 混淆浏览器、Node 和构建时逻辑边界，让代码依赖隐式运行环境。
