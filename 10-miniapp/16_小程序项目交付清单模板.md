# 小程序项目交付清单模板

## 目标

- 提供一份可直接套用的小程序项目交付清单，用于串联开发、自测、体验版验证、提审、灰度发布和回退准备。
- 避免项目在准备上线时，仍通过口头确认或零散消息核对关键交付项。
- 让小程序交付动作从“靠经验记忆”转为“按清单执行”。

## 适用范围

- 适用于微信小程序及后续纳入的同类平台型轻应用的版本交付、自测提审、灰度发布、正式上线和回退准备场景。
- 涉及发布交付要求时，应结合 [15_小程序发布交付规范](15_小程序发布交付规范.md) 使用。
- 涉及平台能力上线影响时，应结合 [14_小程序平台能力接入规范](14_小程序平台能力接入规范.md) 使用。
- 涉及通用工程边界时，应结合 [11_小程序通用规范](11_小程序通用规范.md) 使用。

## 使用说明

- 本模板适合作为版本发布前的执行清单。
- 可按项目实际情况补充“是否适用”“负责人”“完成时间”“备注”等列。
- 不适用项应写明原因，不直接留空。

## 模板

### 1. 基本信息

- 版本号：
- 构建标识：
- 目标宿主平台：微信小程序 / 其它平台型轻应用
- 目标发布形态：体验版 / 灰度版 / 正式版
- 构建基线：
- 发布负责人：
- 验收负责人：
- 计划发布时间：

### 2. 开发完成检查

- [ ] 需求范围已冻结，未完成项已明确记录。
- [ ] 目标宿主平台、环境和发布形态已确认。
- [ ] 登录授权、支付、订阅消息、分享和平台能力改动边界已确认。
- [ ] 相关配置变更、环境开关和接口差异已确认。
- [ ] 发布说明、变更说明或用户提示已补齐。

### 3. 自测与测试检查

- [ ] 关键业务路径已完成自测。
- [ ] 关键冒烟路径已验证。
- [ ] 体验版最小验收路径已验证。
- [ ] 冷启动、热启动和核心页面跳转已验证。
- [ ] 本轮已知问题已汇总并确认是否阻断发布。

### 4. 平台能力检查

- [ ] 登录授权 / 手机号授权能力已验证或确认不适用。
- [ ] 支付能力已验证或确认不适用。
- [ ] 订阅消息 / 分享 / 扫码相关能力已验证或确认不适用。
- [ ] 文件 / 定位 / 相机 / 客服等相关能力已验证或确认不适用。
- [ ] 授权拒绝、用户取消和平台失败场景已验证或确认不适用。

### 5. 构建与版本检查

- [ ] 版本号、构建标识、环境标识和目标平台信息正确。
- [ ] 开发版 / 体验版 / 提审版产物已生成。
- [ ] 构建配置和环境切换结果已验证。
- [ ] 构建日志、版本说明和发布记录已归档。

### 6. 配置与环境检查

- [ ] API 地址、登录环境、支付参数和功能开关符合目标环境。
- [ ] 宿主平台差异配置与环境区分已确认。
- [ ] 敏感配置、支付参数和发布凭据未进入普通日志或源码仓库。
- [ ] 开发、测试、灰度、正式环境未混用。

### 7. 发布前验收检查

- [ ] 首屏可打开并正常启动。
- [ ] 登录链路和核心主流程可用。
- [ ] 支付、分享回流、扫码进入和关键平台能力符合预期或确认不适用。
- [ ] 分包加载、页面切换和关键资源加载符合预期。
- [ ] 缓存、登录态恢复和环境切换行为符合预期。
- [ ] 关键页面或关键流程已完成宿主内体验验收，已确认主动作焦点、信息层级和回流路径达标。
- [ ] 未残留明显 H5 直移、小程序宿主割裂感或只满足提审通过的页面质量问题。

### 8. 发布执行检查

- [ ] 发布窗口时间已确认。
- [ ] 发布负责人、观察人和回退负责人已到位。
- [ ] 提审步骤、发布步骤、灰度扩量或正式放量步骤已确认。
- [ ] 灰度样本范围、停推条件和放量规则已确认。

### 9. 回退准备检查

- [ ] 上一稳定版本记录可用。
- [ ] 灰度停止、版本回退或服务端降级方式已确认。
- [ ] 配置回退、支付开关降级和消息触达降级边界已确认。
- [ ] 故障升级联系人和通知路径已确认。

### 10. 发布后观察检查

- [ ] 启动率、页面打开成功率、关键接口错误率和核心业务指标已开始观察。
- [ ] 登录、支付、订阅消息、分享回流和关键平台能力相关异常已重点观察。
- [ ] 用户反馈入口已准备。
- [ ] 发布结果和后续行动项已记录。

## 输出要求

- 输出交付清单时，应保留完整执行项，不随意压缩为口头描述。
- 如项目已有内部发布单格式，可在保留本模板核心检查项的前提下适配现有格式。
- 如某项检查由自动化系统完成，也应记录对应结果来源。
- 如版本包含首页改版、关键流程重做或演示场景，还应附体验版截图、录屏或等价体验验收材料。
