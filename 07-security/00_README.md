# 目录索引

## 目标

- 为安全目录提供统一入口，先说明哪些场景应该先进入本目录，以及身份、接口、数据、密钥、基础设施和发布安全分别从哪里进入。
- 避免把安全工作压缩成上线前补一份检查单，而忽略设计期、实现期和运行期的持续安全约束。

## 使用说明

- 进入安全规范目录后，默认先阅读本文件，再按任务类型进入对应分册。
- 如任务跨前端、后端、数据库、基础设施或 Electron，先在这里确认共性安全要求，再回到对应技术目录落地。
- 如任务涉及新增能力、接入第三方、开放接口、敏感数据处理、鉴权授权或上线前评审，优先从本目录开始。

## 什么时候先读本页

- 还不确定当前问题属于身份权限、接口暴露、敏感信息、数据隐私、供应链、基础设施还是安全发布问题。
- 需要先判断当前任务应先补安全基线，还是直接回到前端、后端、数据库或 DevOps 目录处理。
- 当前任务跨多个技术目录，但又明显涉及登录、开放接口、敏感数据、凭据或发布放行要求。

## 按任务类型快速入口

- 做账号体系、登录态、权限模型、角色边界或越权控制设计：先读 [01_身份认证与授权安全规范](01_身份认证与授权安全规范.md)。
- 做接口暴露、输入校验、输出脱敏、上传下载、回调验签或开放平台接入：先读 [02_应用与接口安全规范](02_应用与接口安全规范.md)。
- 做密钥、令牌、密码、证书、配置项或环境变量管理：先读 [03_敏感信息与密钥管理规范](03_敏感信息与密钥管理规范.md)。
- 做数据分级、字段脱敏、加密存储、审计留痕或个人信息保护设计：先读 [04_数据安全与隐私保护规范](04_数据安全与隐私保护规范.md)。
- 做依赖治理、镜像来源、供应链审查、漏洞修复或发布前安全检查：先读 [05_依赖与供应链安全规范](05_依赖与供应链安全规范.md)。
- 做主机、容器、网络边界、最小权限、基线加固或云资源暴露治理：先读 [06_基础设施与运行环境安全规范](06_基础设施与运行环境安全规范.md)。
- 做日志审计、异常告警、事件响应、应急处置或复盘留档：先读 [07_安全审计与事件响应规范](07_安全审计与事件响应规范.md)。
- 做需求评审、自测验收、上线门禁或团队对齐：先读 [08_安全评审与发布检查清单](08_安全评审与发布检查清单.md)。

## 与其它目录的配合方式

- 安全目录负责回答“哪些边界要保护、哪些风险要先控、哪些发布前检查不能省略”。
- 涉及接口设计、服务边界和实现细节时，补读 [../03-backend/00_README.md](../03-backend/00_README.md) 或 [../02-frontend/00_README.md](../02-frontend/00_README.md)。
- 涉及敏感字段、库表变更、脱敏和数据生命周期时，补读 [../04-database/00_README.md](../04-database/00_README.md)。
- 涉及发布流程、镜像来源、运行环境和上线放行时，补读 [../06-devops/00_README.md](../06-devops/00_README.md)。
- 涉及验证范围、自测、回归和门禁时，补读 [../05-tests/00_README.md](../05-tests/00_README.md) 和 [../01-general/08_交付质量规范.md](../01-general/08_交付质量规范.md)。

## 安全规范分册

- [01_身份认证与授权安全规范](01_身份认证与授权安全规范.md)：账号认证、会话管理、权限模型、最小授权和越权防控要求。
- [02_应用与接口安全规范](02_应用与接口安全规范.md)：接口输入输出、文件上传下载、回调验签、开放接口和常见应用层风险控制要求。
- [03_敏感信息与密钥管理规范](03_敏感信息与密钥管理规范.md)：密钥、口令、令牌、证书、配置项和密文使用边界要求。
- [04_数据安全与隐私保护规范](04_数据安全与隐私保护规范.md)：数据分级分类、加密脱敏、个人信息处理和审计追踪要求。
- [05_依赖与供应链安全规范](05_依赖与供应链安全规范.md)：三方依赖、制品来源、漏洞修复、镜像治理和供应链审查要求。
- [06_基础设施与运行环境安全规范](06_基础设施与运行环境安全规范.md)：主机、容器、网络、云资源和运行环境的安全基线要求。
- [07_安全审计与事件响应规范](07_安全审计与事件响应规范.md)：日志审计、告警、事件分级、止血、通报、复盘和改进要求。
- [08_安全评审与发布检查清单](08_安全评审与发布检查清单.md)：需求评审、自测、上线前检查和发布门禁可直接复用的清单。

## 阅读顺序建议

- 如需先建立统一安全基线，再按专题落地，建议按 01 至 07 顺序阅读，再用 08 做发布前检查。
- 如任务已经确定为某一技术专题，可先从“按任务类型快速入口”进入，再回看其它相关分册补齐控制项。
- 如任务属于跨端或跨服务协同场景，应先明确身份、数据和密钥边界，再补充接口、依赖和基础设施层控制。

## 不建议的做法

- 只在上线前临时补一轮安全检查，而不在设计和实现阶段提前建立边界控制。
- 把身份、接口、数据、密钥和基础设施问题混成一个“安全”概念处理，不先定位控制面。
- 只看安全目录，不回到前端、后端、数据库、测试或 DevOps 目录落地具体实现和验证。
